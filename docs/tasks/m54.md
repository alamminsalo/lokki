# Milestone 54 — Error Handling & Validation

_Purpose: Implement comprehensive error handling with custom exceptions, graph validation, and configuration validation to improve reliability and debugging._

### Background

Currently, lokki has limited error context and validation:
- Empty `_validate()` method in `FlowGraph` - no graph validation
- Generic exceptions without context
- Limited configuration validation for deployment readiness
- No validation of bucket names, cross-field dependencies

This milestone adds custom exception hierarchy, graph validation, and comprehensive configuration checks.

### Exception Hierarchy

```python
class LokkiError(Exception):
    """Base exception for lokki."""

class GraphValidationError(LokkiError):
    """Raised when flow graph validation fails."""

class ConfigurationError(LokkiError):
    """Raised when configuration is invalid."""

class BuildError(LokkiError):
    """Raised when build process fails."""

class StoreError(LokkiError):
    """Raised when storage operation fails."""

class DeployError(LokkiError):
    """Raised when deployment fails."""
```

### Graph Validation Rules

1. **No empty graphs**: Graph must have at least one entry
2. **Map block integrity**: Map blocks must have inner steps
3. **Reachability**: All steps must be reachable from head

### Configuration Validation Rules

1. **Required fields**: `artifact_bucket` required for deployment
2. **Bucket name**: Valid S3 bucket name format (3-63 chars, lowercase, numbers, hyphens)
3. **Image repository**: Required for image deployments
4. **Lambda constraints**: Timeout 1-900s, Memory 128-10240MB
5. **Batch completeness**: job_queue and job_definition_name must both be set or both empty
