# Milestone 57 — Logging & Observability Enhancements ✅

_Purpose: Enhance logging with correlation IDs, structured events, and performance metrics._

### Background

Logging had good foundation but lacked:
- Correlation IDs for tracing across steps
- Detailed retry attempt information
- Performance metrics logging
- Structured logging events

### Tasks Completed

- [x] **T57.1** Add correlation ID support
  - Added `correlation_id` to `LoggingConfig`
  - Generated unique `correlation_id` for each flow execution
  - Included in all log messages

- [x] **T57.2** Enhance StepLogger
  - Added input/output data size logging
  - Included retry attempt number in retry logs
  - Log exception type and message separately

- [x] **T57.3** Improve MapProgressLogger
  - Added timing details
  - Log average item processing time
  - Added estimated completion time

- [x] **T57.4** Add performance timing utilities
  - Created `@timed` decorator (`lokki/_utils.py`)
  - Created `timing_context()` context manager
  - Automatically logs duration and metrics

- [x] **T57.5** Update JSON log structure
  - Added standard fields: correlation_id, flow_name, run_id, step_name
  - Added timestamp, duration_ms
