# Milestone 51 — Include Configuration for Static Files

## Milestone 51 — Include Configuration for Static Files

_Purpose: Add configuration to include static files (like data/*.parquet) into Docker images for both Lambda and Batch container deployments._

### Background

Currently, Docker images only include:
- Python dependencies from pyproject.toml
- Generated handler code (handler.py, batch.py, batch_main.py)

Users need to include static files like:
- Data files (e.g., `data/*.parquet`, `models/*.pkl`)
- Configuration files
- Any other static assets

This milestone adds an `include` configuration that accepts glob patterns relative to the flow module directory.

### Configuration

The `include` configuration is global per flow project and applies to all container images:

```toml
[include]
paths = ["data/*.parquet", "models/*.pkl", "config/*.json"]
```

Glob patterns are relative to the flow script directory. Only works for container images (`package_type = "image"`), not ZIP packages.

### Files are copied to

- Lambda: `${LAMBDA_TASK_ROOT}/` (i.e., `/var/task/`)
- Batch: Same working directory
