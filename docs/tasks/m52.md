# Milestone 52 — Input Hash Validation for Cache

_Purpose: Add input hash validation to cache system to ensure cached outputs are only used when input data hasn't changed._

### Background

The caching system (implemented in milestone for `--run-id` support) stores step outputs in S3. When checking cache, we need to verify that the input hasn't changed since the cached output was created - otherwise we'd return stale results.

### Implementation Approach

Use S3 object tags to store input hash alongside output:

1. **When writing output**: Compute hash of input, store as S3 tag `input_hash=<hash>`
2. **When checking cache**: Read stored input_hash, compare with current input hash
3. **If mismatch**: Don't use cache, execute step normally

### Behavior

| Scenario | `cache_enabled` | Stored tag | Result |
|----------|------------------|------------|--------|
| Input unchanged | `true` | exists, matches | ✅ Use cache |
| Input changed | `true` | exists, mismatch | ❌ Execute step |
| Old cache (no tag) | `true` | missing | ❌ Execute step (backward compat) |
| No `--run-id` | `false` | N/A | ❌ Skip cache check |

### Code Flow

```python
# In handler:
input_hash = _hash_input(input_data)

# Check cache with input hash validation
stored_hash = store.get_input_hash(flow_name, run_id, step_name)
if (cache_enabled and 
    store.exists(flow_name, run_id, step_name) and
    stored_hash == input_hash):
    return cached_result

# ... execute step ...

# Always write with input_hash tag
output_url = store.write(flow_name, run_id, step_name, result, input_hash=input_hash)
```

### Edge Cases

- **Missing tag**: Treat as hash mismatch (backward compatibility for old cached objects)
- **Input is S3 URL**: Read actual data from S3 before hashing
